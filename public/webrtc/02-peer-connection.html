🔄 WebRTC Example 2: Peer-to-Peer Connection

What this demonstrates: Creating an RTCPeerConnection between two peers. This simulates two users in the same page, but the same code works across different browsers/devices (you'd just need a signaling server to exchange the SDP messages).
How it works:
  1. Peer 1 creates an "offer" (SDP describing their media capabilities)
  2. Peer 2 receives the offer and creates an "answer"
  3. Both exchange ICE candidates (network routes to connect)
  4. Once connected, media flows directly between peers
